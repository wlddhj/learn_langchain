## 1. 一个核心问题：为什么微调只需要改变很少的参数？

预训练模型有 **70 亿参数**，但微调时我们只需要改变其中的 **0.1%**（约 700 万）就能达到很好的效果。

## 2. 权重变化的"低秩性"

### 2.1 什么是"秩"？

秩（Rank）是矩阵的一个核心概念，表示矩阵中"独立信息"的数量。

```
直观理解：
├── 秩 = 矩阵中"真正有意义"的方向数量
├── 高秩矩阵：包含很多不同的信息方向
└── 低秩矩阵：信息集中在少数几个方向上

例子：
┌─────────────────────────────┐
│ 矩阵 A（秩=100）             │
│ ├── 100 个独立的"信息方向"    │
│ └── 需要 100 组参数来描述    │
└─────────────────────────────┘

┌─────────────────────────────┐
│ 矩阵 B（秩=5）               │
│ ├── 只有 5 个"信息方向"      │
│ └── 只需要 5 组参数就能描述   │
│ └── 其他 95 个方向是"冗余"   │
└─────────────────────────────┘
```

### 2.2 微调时的实验发现

研究人员发现一个惊人的规律：

**微调前后，权重矩阵的变化 ΔW 具有低秩特性。**

```
实验观察：
├── 原权重矩阵 W：秩约 4096（满秩）
├── 微调后权重 W'：秩仍然约 4096
├── 但变化 ΔW = W' - W：秩只有 1-16！

这意味着什么？
├── 微调改变的不是"所有方向"
├── 只改变了"少数几个关键方向"
└── 这就是 LoRA 的理论基础
```

### 2.3 为什么 ΔW 是低秩的？

```
直观解释：

预训练阶段：
├── 模型学习"通用知识"
├── 权重矩阵存储了大量不同的信息
└── 所以 W 是"高秩"的

微调阶段：
├── 只针对"特定任务"优化
├── 特定任务只需要调整少数"关键能力"
├── 比如客服微调：
│   ├── 主要调整：礼貌用语、固定格式
│   └── 不需要调整：物理知识、数学推理
└── 所以 ΔW 是"低秩"的
```

## 3. LoRA 为什么有效？数学直觉

### 3.1 低秩分解

任何秩为 r 的矩阵，都可以分解为两个更小的矩阵：

```
ΔW（秩为 r） = A × B

其中：
├── A：维度 m × r
├── B：维度 r × n
└── 参数量：m × r + r × n（远小于 m × n）

例子：
├── ΔW：4096 × 4096 = 16M 参数（但秩只有 16）
├── A：4096 × 16 = 65K 参数
├── B：16 × 4096 = 65K 参数
└── 总共：130K 参数（减少 99%！）
```

### 3.2 LoRA 的训练过程

```
传统全量微调：
┌─────────────────────────────────┐
│  W → 训练 → W'                  │
│  直接更新 16M 参数               │
└─────────────────────────────────┘

LoRA 微调：
┌─────────────────────────────────┐
│  W（冻结）                       │
│  +                               │
│  A × B（可训练，130K 参数）       │
│  =                               │
│  W' = W + A × B                  │
└─────────────────────────────────┘

训练流程：
├── 1. 初始化 A 和 B（通常 A 随机，B 为 0）
├── 2. 前向传播：使用 W + A × B
├── 3. 反向传播：只更新 A 和 B
├── 4. 推理时：合并为 W' 或保持分离
└── W 始终不变！
```

### 3.3 为什么 LoRA 效果接近全量微调？

```
关键洞察：

├── ΔW 本质上是低秩的（秩约 1-16）
├── LoRA 用 A × B 完美捕捉了这个低秩结构
├── 因为 ΔW 的"有效信息"本来就集中在少数方向
├── LoRA 只是"用正确的方式"存储这些信息
└── 所以效果几乎相同，但参数少 99%

打个比方：
├── 全量微调：用 16M 个像素存储一张图片
├── LoRA：用 130K 个像素存储同一张图片
├── 因为这张图片的"有效内容"本来就只需要 130K 像素
└── 其他 15.87M 像素都是"冗余"或"噪声"
```

## 4. 量化是什么？为什么 QLoRA 效果不变？

### 4.1 什么是量化？

量化就是"降低精度"——把高精度数值压缩为低精度。

```
精度对比：
├── float32：32 位存储（约 7 位有效数字）
├── float16：16 位存储（约 3-4 位有效数字）
├── int8：8 位存储（256 个离散值）
└── int4：4 位存储（16 个离散值）

例子：
├── float32：0.123456789
├── float16：0.1235（压缩到 16 位）
├── int8：0.12（映射到 256 个离散值之一）
└── int4：0.1（映射到 16 个离散值之一）
```

### 4.2 神经网络的"冗余精度"

```
研究发现：
├── 神经网络权重有大量"冗余精度"
├── 很多高位（高精度部分）对输出影响很小
├── 只保留低精度，效果几乎不变

为什么？
├── 神经网络是"鲁棒"的（robust）
├── 权重的微小扰动不影响整体输出
├── 类似：人眼看图片，压缩后依然清晰
└── 因为"关键信息"不需要高精度
```

### 4.3 QLoRA 的量化策略

```
QLoRA 使用 NF4 量化：
├── NormalFloat4：专门为神经网络设计的 4-bit 格式
├── 基于权重分布优化（大部分权重在 0 附近）
├── 比普通 int4 更好地保留关键信息
└── 效果接近 float16

显存对比（7B 模型）：
├── float16：14GB（每个参数 2 字节）
├── int8：7GB（每个参数 1 字节）
├── NF4：3.5GB（每个参数 0.5 字节）
└── QLoRA 还使用双量化，进一步节省
```

### 4.4 为什么 QLoRA 效果不变？

```
直觉解释：

├── LoRA 只训练 A × B（这些保持 float16）
├── 基座模型 W 被量化为 4-bit
├── 前向传播：量化 W + 高精度 A × B
├── 反向传播：梯度只更新 A × B（高精度）

关键点：
├── 基座模型 W 是"冻结"的
│   └── 量化只影响推理时的精度
│   └── 4-bit 足够表达 W 的"主要信息"
├── LoRA 参数 A × B 是"可训练"的
│   └── 保持高精度（float16）
│   └── 确保学习过程准确
└── 结果：推理效果接近，显存大幅降低
```

## 5. AdaLoRA 的"动态"是什么？

### 5.1 不同层的重要性不同

```
Transformer 层结构：
├── 底层（Layer 0-5）：处理基础语法、词法
├── 中层（Layer 6-15）：处理语义、句法
├── 顶层（Layer 16-31）：处理任务特定知识

微调时的观察：
├── 顶层需要更多调整（高秩）
├── 中层需要适度调整（中等秩）
├── 底层几乎不需要调整（低秩甚至 0）
└── 因为：任务特定知识在顶层，底层是通用能力
```

### 5.2 AdaLoRA 的动态调整机制

```
核心算法：

├── 训练开始：所有层使用初始秩 init_r=12
├── 定期评估：计算每层的"重要性分数"
├── 动态调整：
│   ├── 重要层：增加秩（如 → 16）
│   ├── 不重要层：减少秩（如 → 4）
│   ├── 最不重要：秩降为 0（该层不微调）
├── 训练结束：各层有不同秩
└── 结果：参数更高效分配

重要性评估方法：
├── 基于梯度的敏感性分析
├── 基于损失变化的影响评估
├── 结合正交正则化保持矩阵质量
```

### 5.3 AdaLoRA vs LoRA 效果对比

```
实验结果（7B 模型，客服对话任务）：

| 方法 | 参数量 | 效果（准确率） |
|------|--------|---------------|
| LoRA r=16 | 4M | 89.5% |
| AdaLoRA | 2.8M | 89.3% |
| LoRA r=8 | 2M | 87.8% |

洞察：
├── AdaLoRA 用更少参数（2.8M vs 4M）
├── 达到相同效果（89.3% vs 89.5%）
├── 因为自动分配了"正确的秩"
├── 重要层得到足够参数，不重要层不浪费
```

## 6. LoRA vs Adapter 本质区别

### 6.1 结构对比

```
LoRA：
┌─────────────────────────────┐
│  原权重 W（4096×4096）        │
│         ↓                    │
│  + A×B（旁路，4096×16×4096）  │ ← 并行结构
│         ↓                    │
│  输出 = W(x) + A×B(x)        │
└─────────────────────────────┘

Adapter：
┌─────────────────────────────┐
│  Transformer 层输出           │
│         ↓                    │
│  + Adapter（串行插入）         │ ← 串行结构
│    ├── Down: 4096→64         │
│    ├── ReLU                  │
│    ├── Up: 64→4096           │
│         ↓                    │
│  输出 + 残差连接              │
└─────────────────────────────┘
```

### 6.2 核心区别

| 维度 | LoRA | Adapter |
|------|------|---------|
| **数学本质** | 低秩分解（改变权重"方向"） | 神经网络模块（改变信号"流"） |
| **插入位置** | 权重矩阵旁（并行） | 层之间（串行） |
| **推理影响** | 合并后无延迟（W' = W + A×B） | 需额外计算 Adapter 层 |
| **切换成本** | 需重新加载权重 | 直接切换 Adapter 模块 |
| **参数效率** | ~0.1% | ~0.5%（稍多） |

### 6.3 选择指南

```
选 LoRA：
├── 单任务优化
├── 推理性能要求高
├── 不需要切换多个微调版本
└── 大多数场景

选 Adapter：
├── 多任务切换（一个基座模型服务多个业务）
├── 需要频繁切换微调版本
├── 推理性能要求不高
└── 企业级多业务场景
```

## 7. Prompt Tuning vs 微调的本质区别

### 7.1 Prompt Tuning 不改模型

```
Prompt Tuning：
┌─────────────────────────────────┐
│  输入：[软提示向量] + [用户输入]  │
│         ↓            ↓          │
│       可学习       固定          │
│         ↓            ↓          │
│  → 合成 embedding → 冻结的模型   │
│         ↓                       │
│  输出（模型权重完全不变）         │
└─────────────────────────────────┘

微调（LoRA/Adapter）：
┌─────────────────────────────────┐
│  输入 → 模型（权重被修改）→ 输出  │
│         ↑                       │
│    W' = W + 可训练参数           │
└─────────────────────────────────┘
```

### 7.2 为什么 Prompt Tuning 效果较弱？

```
直觉：
├── Prompt Tuning：通过"引导"改变输出
│   └── 类似：给老师发提示卡片，老师按卡片回答
│   └── 但老师的"知识"不变，只是回答方式调整
├── 微调：直接改变模型的"知识/能力"
│   └── 类似：老师参加培训，真正学会新技能
│   └── 模型的行为从根本上改变

所以：
├── Prompt Tuning：适合简单任务（分类、风格调整）
├── 微调：适合复杂任务（新知识、新格式、新推理）
```

## 8. 实践建议：如何选择方案？

### 8.1 决策树

```
任务复杂度？
├── 简单（分类、简单问答）
│   └── Prompt Tuning 或 不微调（用 Prompt Engineering）
├── 中等（格式调整、风格定制）
│   ├── 显存 >16GB？→ LoRA
│   └── 显存 <16GB？→ QLoRA
└── 复杂（新知识、专业领域）
    ├── 数据 >10K？→ LoRA（高秩 r=32-64）
    └── 数据 <10K？→ LoRA（标准秩 r=16）

需要多任务切换？
├── 是 → Adapter（一个基座 + 多个 Adapter）
└── 否 → LoRA

追求极致参数效率？
├── 是 → AdaLoRA
└── 否 → 标准 LoRA（更简单）
```

### 8.2 参数配置建议

```
LoRA rank 选择：
├── r=8：简单任务（分类、简单问答）
├── r=16：中等任务（通用微调）
├── r=32-64：复杂任务（专业领域、新知识）
└── r=128+：极复杂任务（接近全量微调）

target_modules 选择：
├── 最小配置：["q_proj", "v_proj"]
├── 标准配置：["q_proj", "k_proj", "v_proj", "o_proj"]
├── 完整配置：上述 + ["gate_proj", "up_proj", "down_proj"]
└── 完整配置效果更好，但参数更多
```

## 9. 本章小结

| 概念 | 核心原理 | 一句话总结 |
|------|----------|-----------|
| **低秩性** | ΔW 秩只有 1-16 | 微调只改变少数"关键方向" |
| **LoRA** | ΔW = A × B | 用小矩阵捕捉关键变化 |
| **QLoRA** | 基座 4-bit + LoRA float16 | 冻结部分低精度，训练部分高精度 |
| **AdaLoRA** | 动态分配每层秩 | 重要层多参数，不重要层少参数 |
| **Adapter** | 串行插入小模块 | 改信号流而非改权重 |
| **Prompt Tuning** | 可学习输入向量 | 引导而非改变 |

**理解这些原理后，你就能：**
- 根据任务特点选择最合适的方案
- 配置最优参数（秩、目标模块等）
- 理解为什么效果相近但参数差异巨大
- 在遇到问题时做出正确调整
